## Project Overview

Marketing website for **ECOVALUE** — a Belgian certified energy inspector (erkend energiedeskundige type A) offering EPC certificate services across Flanders. The site is fully in Dutch (nl-BE).

- **Stack:** Next.js 16 (App Router), React 19, TypeScript (strict), Tailwind CSS v4, shadcn/ui (base-nova style)
- **Hosting target:** Vercel (static export + server routes)
- **Purpose:** SEO-driven lead generation — each page targets specific Dutch search queries around EPC certification

## Commands

```bash
npm run dev      # Start dev server (Turbopack)
npm run build    # Production build
npm run start    # Serve production build
npm run lint     # ESLint
```

No test suite is configured.

## Architecture

```
src/
  app/                  # Next.js App Router — one folder = one route
    api/indexnow/       # POST endpoint to submit URLs to IndexNow
    epc-attest/[stad]/  # Programmatic city pages (12 cities, generateStaticParams)
    sitemap.ts          # Dynamic XML sitemap
    layout.tsx          # Root layout: fonts, global schema, Navbar/Footer
    globals.css         # Design tokens + Tailwind theme
  components/
    layout/             # Navbar, Footer
    sections/           # Reusable page sections (Hero, PricingTable, Breadcrumb…)
    seo/                # SchemaOrg injector + schema builder functions
    ui/                 # shadcn primitives (button, card, accordion, etc.)
  lib/
    constants.ts        # Single source of truth: BUSINESS info, PRICES, EPC_LABELS, NAV_LINKS, COVERAGE_CITIES
    utils.ts            # cn() helper (clsx + tailwind-merge)
```

**Key pattern:** All business data (name, phone, prices, promises) lives in `src/lib/constants.ts`. Pages import from there — never hardcode business details inline.

**Schema pattern:** `src/components/seo/schemas.ts` exports builder functions (`buildFaqSchema`, `buildBreadcrumbSchema`, `buildServiceSchema`). Inject via `<SchemaOrg schema={...} />` in the page JSX. The root `localBusinessSchema` is injected once in `layout.tsx`.

**City pages:** `src/app/epc-attest/[stad]/page.tsx` generates 12 static city pages via `generateStaticParams`. The `cityData` record is the authoritative list — add new cities there and in `sitemap.ts`.

## Code Conventions

- `@/` alias maps to `src/` — always use it for imports
- TypeScript strict mode is on; no `any`, no implicit returns
- Tailwind v4 with CSS custom properties — use design tokens from `globals.css` (`text-ink`, `bg-cream`, `text-blue`, `text-gold`, etc.) not raw hex values
- Font classes: `font-sans` (Inter), `font-heading` (Fraunces), `font-mono` (JetBrains Mono)
- Responsive sizing via `clamp()` for headings, `sm:` breakpoint for layout shifts
- shadcn components in `src/components/ui/` — add new ones with `npx shadcn add <component>`
- Icons from `lucide-react` only

## External Services & Integrations

- **IndexNow API** (`/api/indexnow`): POST `{ urls: string[] }` to trigger Bing/Yandex indexing. Key is set via `INDEXNOW_KEY` env var; the key verification file lives at `public/a1b2c3d4e5f67890abcdef1234567890.txt`
- **Google Fonts**: Inter, Fraunces, JetBrains Mono — loaded via `next/font/google` in `layout.tsx`
- **Schema.org**: JSON-LD injected server-side via `<SchemaOrg>` — no third-party schema library

## What NOT To Do

- Do not edit files in `.next/` — build output, auto-generated
- Do not hardcode business info (name, phone, prices, address) outside `src/lib/constants.ts`
- `trailingSlash: false` is intentional — do not add trailing slashes to internal `<Link href>` values
- Redirects in `next.config.ts` are permanent (301) — changing destinations breaks SEO; discuss before modifying
- Do not add the `epc-waarde-berekenen` page as a new route — it redirects to `/epc-waarden-tabel` (see `next.config.ts`)
